# Garden Planner — Design Spec
_2026-04-14_

## Overview

A React v19 single-page app that lets a user configure a vegetable garden plot, select plants filtered by light condition, view a scaled top-down garden layout, and consult a growing/harvest calendar. All plant data is sourced exclusively from the two PDFs in `docs/` and extracted at build time.

---

## Architecture

**Two distinct phases:**

**Build-time data extraction** — a standalone Node script (`scripts/extract-plants.js`) runs `pdfjs-dist` against both PDFs and writes `src/data/plants.json`. This script is a dev tool only; its output is committed to source. It is not part of the app bundle.

**Runtime app** — a pure client-side Vite + React v19 SPA. No API calls, no PDF parsing at runtime. Deployed as static files.

```
src/
  data/plants.json            ← generated by script, committed
  components/
    Sidebar.jsx               ← plot dims + light filter + plant checklist
    GardenLayout.jsx          ← scaled top-down grid
    GrowingCalendar.jsx       ← 12-month grid per plant
  App.jsx                     ← layout shell, state, URL sync
scripts/
  extract-plants.js           ← one-time PDF extraction tool
docs/
  EHT-077-texas-home-vegetable-gardening-guide.pdf
  VegetableGardenPlantingGuide-Jan2025-English.pdf
```

**Styling:** Tailwind v4 (no PostCSS plugin required with React v19).

---

## Data Model

`src/data/plants.json` shape:

```json
[
  {
    "name": "Tomato",
    "light": "direct",
    "rowSpacingIn": 18,
    "inRowSpacingIn": 24,
    "daysToHarvest": "70–85 days",
    "calendar": {
      "sow":     [false, false, true,  true,  false, false, false, false, true,  false, false, false],
      "grow":    [false, false, false, true,  true,  true,  false, false, false, true,  false, false],
      "harvest": [false, false, false, false, true,  true,  true,  false, false, false, true,  false]
    }
  }
]
```

- `light` — one of `"direct"`, `"partial"`, `"low"`
- `rowSpacingIn` — inches between rows; drives garden layout row heights
- `inRowSpacingIn` — inches between plants within a row; used to calculate plant count per row
- `daysToHarvest` — free-form string (e.g. `"70–85 days"`, `"60–70 days from transplant"`)
- `calendar` arrays — 12 booleans, index 0 = January; a month may be true in multiple phases

> **Note:** PDF text extraction is imperfect. The generated `plants.json` requires a manual review pass before use.

## Component Structure & State

All state lives in `App.jsx` and flows down as props. No context or external state library.

```
App
├── state: widthIn, lengthIn, lightFilter, selectedPlants[]
│
├── Sidebar
│   ├── DimensionInputs     ← width + length number inputs (inches); shows calculated sq ft
│   ├── LightFilter         ← three toggle buttons: Direct / Partial / Low
│   └── PlantChecklist      ← plants filtered to lightFilter, checkbox per plant
│
└── MainPanel
    ├── GardenLayout        ← receives widthIn, lengthIn, selectedPlants[]
    └── GrowingCalendar     ← receives selectedPlants[]
```

**Page layout:** Left sidebar (fixed width) with controls. Right main panel stacks GardenLayout above GrowingCalendar.

---

## URL State Persistence

On every state change, `App` calls `history.replaceState` to sync inputs to the URL query string. On mount, URL params initialize state. No router library required.

**URL format:**
```
?w=40&l=80&light=direct&plants=Tomato,Pepper,Squash
```

- `w` — plot width in inches
- `l` — plot length in inches
- `light` — `direct`, `partial`, or `low`
- `plants` — comma-separated plant names (URL-encoded)

Invalid or missing params fall back to defaults silently: `w=40`, `l=80`, `light=direct`, no plants selected.

## Garden Layout

**Visual style:** Scaled top-down grid. Container aspect ratio matches `widthIn / lengthIn` so the plot is always proportional on screen.

**Row calculation (greedy, top-to-bottom):**
- Iterate selected plants in checklist order
- For each plant, add rows of `rowSpacingIn` height until the remaining plot length is exhausted
- Row height as percentage of container: `(rowSpacingIn / lengthIn) * 100%`
- Plant count per row: `Math.floor(widthIn / inRowSpacingIn)`, minimum 1
- Multiple rows of the same plant are added until the next plant's rows begin

**Visual details:**
- Each plant has a consistent color (derived from its index in the plants list)
- Each row shows: plant name, row spacing, plant count (e.g. "Tomato · 18″ · 2 plants")
- Rulers: top edge (width in feet), left edge (length in feet)
- Remaining space label below grid (e.g. "2″ unused")

---

## Growing Calendar

**Layout:** 13-column CSS grid — plant name column + 12 month columns (Jan–Dec).

**Per plant:** Three sub-rows stacked — Sow / Grow / Harvest. Each cell is colored when that phase is active for that month, gray otherwise.

**Colors:**
- Sow: light green (`#c8e6c9`)
- Grow: medium green (`#a5d6a7`)
- Harvest: dark green (`#66bb6a`)

`daysToHarvest` is shown as a small label at the end of the Harvest sub-row.

Legend at the bottom: Sow · Grow · Harvest color swatches.

---

## Edge Cases

| Condition | Behavior |
|-----------|----------|
| Plot too small for any row | Empty grid + "Plot too small for selected plants" message |
| Selected plants overflow plot | Fitting rows shown in grid; overflow plants listed below in a muted "doesn't fit" section |
| No plants selected | Layout and calendar show empty states |
| Light filter removes all selected plants | Selections cleared; empty state shown |
| Invalid URL params on load | Silent fallback to defaults |
| In-row spacing wider than plot width | Plant count per row = 1 (minimum) |
